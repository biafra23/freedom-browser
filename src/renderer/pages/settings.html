Networks

The chains Freedom resolves names and balances on, and the RPC / prover endpoints behind them. Each network picks a verification strategy; endpoint sources supply the URLs it uses.

Endpoint sources

RPC and prover endpoints grouped by chain. Disable a built-in source to stop using it, add your own node, or configure an API key for a keyed provider.

Resolution method

How Freedom resolves .eth and .box names to the content they point at. Colibri verifies each answer cryptographically against the Ethereum sync committee; the public-RPC quorum cross-checks several RPC providers; custom RPC trusts a node you run yourself.

Each answer is verified against the Ethereum sync committee via a partner-hosted prover. Stronger than RPC cross-checking. Adds a single trust dependency on the prover; fall back to the public-RPC quorum if it fails.

Ask several public Ethereum RPCs in parallel and only trust answers where they agree. The previous Freedom default.

Resolve via a single Ethereum node you run yourself. No cross-checking, no prover dependency — you trust your own node.

Fall back to public-RPC quorum if primary fails

When the primary method errors or can't verify, retry via the public-RPC quorum rather than surfacing the failure. The fallback is announced in DevTools so primary-method health regressions stay visible.

Partner-hosted Colibri prover. The default is operated by corpus.core; you can point at any compatible prover endpoint.

Use ZK consensus proof

Bootstraps the sync committee via a recursive zero-knowledge proof instead of classical light-client updates. Smaller bootstrap, partner-recommended on.

Public-RPC quorum

Used as the primary path when "Public-RPC quorum" is selected above, and as the fallback for the other methods. Freedom asks multiple servers and only trusts answers where several agree.

Block unverified ENS navigation

Show a warning page when only one RPC answers (no cross-check possible). Turn off to navigate directly with only the amber shield indicator.

Cross-check across multiple public RPCs

Advanced quorum parameters

Providers per wave (K)

Number of RPCs queried in parallel per lookup. Minimum 3 for verified outcomes.

Agreement threshold (M)

How many providers must return byte-identical answers. Must be ≤ K.

Per-call timeout (ms)

Slow providers are skipped after this time.

Block anchor

All providers query at this block. latest is ~1.5 min behind head; finalized is ~12 min behind but strongest.

Anchor cache TTL (ms)

How long one pinned block is reused across resolutions.

Public RPC providers

Edit the list of Ethereum RPCs Freedom uses for cross-checking. Fewer than 3 entries means every resolution will be marked unverified.
